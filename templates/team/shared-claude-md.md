# CLAUDE.md — チーム開発共通設定

## コーディング規約

- PR は機能単位で小さく保つ（レビュー可能なサイズ）
- 変数名・関数名は英語。コメントは日本語でも可
- マジックナンバーは定数化する
- TODO コメントには担当者と日付を添える: `// TODO(yamada, 2026-04-01): ...`

## ブランチ戦略

- `main`: 本番環境。直接pushは禁止
- `develop`: 開発環境へのデプロイブランチ
- `feature/{issue-number}-{brief-description}`: 機能開発
- `fix/{issue-number}-{brief-description}`: バグ修正
- `hotfix/{brief-description}`: 緊急の本番修正

## PR・コードレビュールール

- セルフレビューをしてからレビュー依頼する
- レビュー依頼から24時間以内にレビューする（チーム合意が必要）
- `LGTM` のみのコメントは避け、承認の根拠を一言添える
- 破壊的変更（APIの変更・DBスキーマ変更）は必ずチーム全員に共有してから進める

## コミットメッセージ

```
{type}: {変更の概要（命令形）}

{なぜこの変更をしたかの背景（任意）}
```

type一覧: `feat` / `fix` / `refactor` / `test` / `docs` / `chore`

## セキュリティ

- `.env` はコミットしない（`.env.example` のみコミット）
- APIキー・シークレットはシークレットマネージャーで管理する
- `console.log` でシークレット値を出力しない

## Claude Codeへの指示

- テストがない場合は実装前にテスト方針を確認する
- チームの既存コードスタイルに合わせる（勝手にリファクタしない）
- 共有システムへの操作（デプロイ・DB変更等）は実行前に確認する
